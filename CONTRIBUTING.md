## Branch Names

Format: `<username>/<slug>`

`<username>` represents your GitHub username

`<slug>` represents a very short string describing the change

## Commit Messages

Format: `<type>(<scope>): <subject>`

`<scope>` is optional

Example `<type>` values:

- feat
- fix
- build
- chore
- ci
- docs
- style
- refactor
- perf
- test
